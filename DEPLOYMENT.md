# PCG Platform — Deployment Guide

## Quick Deploy to Render

### 1. Push to GitHub
```bash
git init && git add . && git commit -m "Initial production deploy"
git remote add origin https://github.com/YOUR_USER/pcg-platform.git
git push -u origin main
```

### 2. Deploy via Render Blueprint
1. Go to **dashboard.render.com → New → Blueprint**
2. Connect your GitHub repo
3. Render will read `render.yaml` and create 3 resources: `pcg-db`, `pcg-api`, `pcg-web`

### 3. After first deploy — fix CORS (REQUIRED)
The most common post-deploy issue. Do this immediately:

1. In Render Dashboard → **pcg-api** → Environment
2. Find `CORS_ORIGINS`
3. Set it to your exact **pcg-web URL** (e.g. `https://pcg-web-abc123.onrender.com`)  
   ⚠️ No trailing slash. Copy-paste from the pcg-web service page.
4. Click **Save** — Render will redeploy pcg-api automatically.

### 4. Verify everything works
Visit: `https://pcg-api.onrender.com/health`

Expected response:
```json
{
  "status": "ok",
  "checks": { "database": "ok" },
  "cors_origins": ["https://pcg-web-abc123.onrender.com"]
}
```

---

## If NEXT_PUBLIC_API_URL is wrong
If Render assigned a different URL to pcg-api (e.g. `pcg-api-xyz.onrender.com`):

1. Render Dashboard → **pcg-web** → Environment
2. Set `NEXT_PUBLIC_API_URL` = `https://pcg-api-xyz.onrender.com/api/v1`
3. Click Save → Render will rebuild pcg-web (Next.js bakes this at build time)

## Free Tier Limitations
- **Spin-down**: Services sleep after 15 min of inactivity. First request takes 30–60 s.  
  The frontend now shows "API server is waking up…" instead of a cryptic JSON error.
- **No background workers**: Metrics worker runs inline (`METRICS_WORKER_INLINE=true`)
- **512 MB RAM**: Sufficient for this workload

## Local Development
```bash
# 1. Install deps
npm install

# 2. Copy env and fill in values
cp apps/api/.env.example apps/api/.env

# 3. Start Postgres (Docker)
docker run -d --name pcg-pg -e POSTGRES_PASSWORD=postgres -e POSTGRES_DB=pcg_dev -p 5432:5432 postgres:15

# 4. Run migrations
cd apps/api && npx prisma migrate dev && cd ../..

# 5. Start everything
npm run dev
```

Frontend: http://localhost:3000  
API: http://localhost:4000  
API docs: http://localhost:4000/health

## Environment Variables Reference

### pcg-api (required)
| Variable | Value | Notes |
|---|---|---|
| `DATABASE_URL` | auto | Set by Render from pcg-db |
| `JWT_SECRET` | auto | Generated by Render |
| `JWT_REFRESH_SECRET` | auto | Generated by Render |
| `CORS_ORIGINS` | your pcg-web URL | **Must set manually** |
| `ADMIN_EMAILS` | comma-separated emails | Gets admin UI access |
| `METRICS_WORKER_INLINE` | `true` | Required on free tier |

### pcg-web (required)
| Variable | Value | Notes |
|---|---|---|
| `NEXT_PUBLIC_API_URL` | your pcg-api URL + `/api/v1` | Baked at build time |
